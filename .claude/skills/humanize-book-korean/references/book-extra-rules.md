# 한국 IT 책 톤 — humanize-korean 보강 규칙

`references/quick-rules.md`(im-not-ai)의 A~J 카테고리에 더해, 한국 IT 책 작업 중 발견된 패턴을 카테고리 P~T로 보강합니다. 본 파일은 `humanize-korean` 룰북과 함께 로드되어 한국 IT 책에 특화된 윤문을 가능하게 합니다.

원본 룰북의 Do-NOT(고유명사·수치·인용·법률 조문·영어 약어)는 그대로 유지합니다.

## Q. 객체·시스템에 "책임" 부여 (영어 직역)

한국에서 "책임"은 도덕적·법적 책임 의미가 강합니다. 객체·코드·모듈에 "is responsible for" 직역 금지.

| ID | 패턴 | 심각도 | 처방 |
|---|---|---|---|
| Q-1 | "X가 Y를 책임진다" (X = 객체·모듈·시스템) | S1 | "X가 Y를 담당합니다 / X가 Y를 합니다 / X의 역할은 Y입니다" |
| Q-2 | "각각 무엇을 책임지는지" | S1 | "각각 어떤 역할을 하는지" |
| Q-3 | "이 클래스의 책임" | S1 | "이 클래스가 하는 일 / 이 클래스의 역할" |
| Q-4 | "책임에 맞게 분리" | S1 | "역할에 맞게 분리 / 용도에 맞게 분리" |

**예외 (보존)**: 사람·부서·팀이 책임을 지는 표현은 자연스러우므로 보존. 예: "행정 처분의 책임은 사람이 진다", "운영팀이 장애 대응 책임을 진다".

## 적용 우선순위

본 파일의 패턴은 `quick-rules.md`(A~J)와 **함께** 적용됩니다. 우선순위:

1. **A~J** (im-not-ai 원본): 일반 한국어 AI 티 패턴
2. **P~T, V, X** (본 파일): 한국 IT 책 특화 패턴
3. **U** (본 파일): 도메인별 패턴 (examples/{domain}/glossary.md로 분리 가능)
4. **W** (본 파일): 책 메타 영역 (공저 노트)
5. **Y** (본 파일): 기존 카테고리 정규식 보강

P~T·V·X는 모든 한국 IT 책에 공통 적용. U는 도메인별. examples/spring-ai-book/domain-glossary.md 가 본 책 사례.

## 개선 사이클

본 파일은 계속 발전합니다. 사용자가 새 패턴을 짚으면:

1. 위 카테고리 중 적절한 곳에 ID 추가 (P-6, Q-5 등)
2. examples/{domain}/playbook.md 에 Before/After 사례 추가
3. metrics.py 또는 metrics_v2.py 에 측정 함수 추가 (정량 검출용)
4. CHANGELOG.md 에 항목 추가

스킬 사용자의 PR도 환영합니다.
