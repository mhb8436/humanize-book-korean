# 한국식 톤 규칙 (본 서적 전체 적용)

**Why:** 2026-05-27 사용자 피드백. 초안 본 서적이 "출판사 빠꾸 우려" 수준으로 LLM 응답스러웠고, "본 책의 정체성이다" 같은 영어식·메타 표현이 만연했다. 사용자 자바책(2014년 출간 직접 집필)을 기준 톤으로 잡고, 머리말을 4번 다듬어 합의된 규칙. 본 규칙은 본 서적 26개 파일 전부에 적용한다.

**How to apply:** 본 서적의 어느 챕터·부록·앞뒷부속을 손볼 때마다 이 규칙을 적용한다. 새 챕터를 쓸 때도 처음부터 본 규칙으로 작성. 작업 끝마다 아래 grep 점검을 통과해야 한다.

## 1. 종결어미

- **존댓말 ~습니다체 일관 적용**
  - 했다 → 했습니다 / 이다 → 입니다 / 한다 → 합니다 / 된다 → 됩니다
  - 사용자 자바책은 ~다체이지만 이 책은 사용자 결정으로 ~습니다체.
- 청유형은 **~봅시다 / ~봅니다 / ~겠습니다** (강의 톤). "~보자"는 너무 격의 없음.

## 2. 금지 표현 (영어식·LLM 메타체)

### 비유 명사 — 절대 금지
- ~의 자리 / ~한 자리 / ~ 자리에 있다
- ~의 흐름 / ~의 교차점
- ~의 토대 / ~ 위에 서 있다
- ~의 정체성
- ~의 약속 / 약속하는 ~가지
- 한 페이지 한 페이지가 / 한 자리에 모인다
- ~한 자산이다 / ~한 디딤돌이다

### 메타·요약체 — 절대 금지
- 한 줄 결론 / 한 줄로 요약하면 / 다른 말로 표현하면
- 본 ~의 가장 큰 ~ 가치다
- 본 ~의 핵심 / ~ 정수
- 본 ~의 결정적 차이
- 한 줄로 / 한 페이지에 / 한 자리에서
- 본 ~ 한 자리

### 인사·환영·끝맺음 멘트 — 절대 금지
- 함께 시작합시다 / 함께 시작해 보시죠
- 다음 페이지에서 만나자 / 다음 페이지부터 본문이 시작됩니다
- 이건 내 일이다 느끼시는 순간이 ~ 자리입니다
- ~ 가장 잘 작동하는 자리입니다
- 본 ~을 펴 든 당신에게

### 호칭
- **본 서적 → 이 책**
- **본 챕터 → 이 장**
- **본 부록 → 이 부록**
- **본 절 → 이 절**
- 한 챕터 안에서 "이 책" 반복되면 가끔 생략 가능

## 3. 영어식 구조 — 한국식으로

- 수동태 "~되었습니다" → 능동태 "~하였습니다 / ~했습니다 / ~로 작성했습니다"
- "~의 X-Y" 영어식 → "X-Y한 ~" 한국식
- em-dash (—) 거의 사용 안 함. 한국어 쉼표(,), 괄호(), 마침표(.)로 분리
- "Spring AI는 ~를 제공한다 — 사업자 종속에서 ~" → "Spring AI는 사업자에 종속되지 않은 ~를 제공합니다"
- "~의 ~이라는 ~" 식 영어 명사구 직역 회피

## 4. 글머리표·헤딩

### 헤딩 — 사실 헤딩만
- "이 책의 자리" → "이 책의 구성" / "이 책의 배경"
- "본 챕터가 답하는 것" → "이 장에서 다루는 내용"
- "페르소나와 학습 방식" → "이 책을 읽는 방법"
- "본 챕터의 한 줄 결론" → 헤딩 자체 삭제
- 메타 헤딩 안 씀. 콘텐츠 자체 명사로.

### 글머리표 — 정보 나열에만
- "필수: ~ / 권장: ~ / 환경: ~" 메타 라벨 금지
- 정보 항목은 OK ("Spring Boot 3.5.6 / Java 17 / Spring AI 1.0.x")
- 한국 IT 도서는 글머리표보다 풀어 쓰는 단락을 선호

## 5. 사용자 책 톤의 본질

사용자 자바책 1·5·6·7·11·13장 분석 결과:

- **사실로 시작**: "자바는 객체지향언어이다."
- **장 도입 직설적**: "이번 장에서는 ~를 다룬다."
- **연결어 단순**: 즉 / 이때 / 이는 / 다음의 / 또한 / 물론 / 그렇다면
- **저자 1인칭 자연스럽게**: "공부해보자 / 살펴보자 / 가정하자" (이 책은 ~봅시다)
- **문장은 길어도 사실로만 채움**
- **인사·환영·끝맺음 멘트 0건**
- **추상 명사 0건** — "자리/흐름/정체성/약속" 한 번도 등장 안 함

## 6. 9가지 잠재 영어체 패턴 (2026-05-27 추가)

ch01 점검 중 발견된 추가 영어체 패턴. 본문 16장·부록 5종에서 이 9가지를 모두 확인해 제거한다.

### A. 강조성 숫자·도구 비유 — 금지
- "화이트보드 한 장으로 설명할 수 있다" → "설명할 수 있다 / 안다"
- "30분 안에 머릿속에 잡아야 할" → "30분 정도 읽어 두면 좋은"
- "한 페이지로 ~한다 / 한 줄로 정리하면"

### B. 동작 비유 — 개념의 사물화·의인화 금지
- "이 다섯 가지가 잡히면" → "다섯 가지를 알면"
- "후속 15장이 매끄럽게 흐른다" → "나머지 15장을 쉽게 공부할 수 있다"
- "머릿속에 잡아야 할 전체 그림" → "큰 그림"
- "손에 쥔다" → "안다 / 배운다"
- "RFP가 떨어진다" → "RFP가 들어온다 / 도착한다"
- "머릿속에 박힌다 / 잡힌다 / 정리되어 있으면"

### C. 화자·태도 강조 — 한국 사람은 안 씀
- "본인 입으로 설명할 수 있다" → "설명할 수 있다"
- "손으로 확인한다 / 만진다" → "확인한다 / 다룬다"
- "직접 ~한다" — 가끔은 OK이지만 반복되면 제거

### D. 가치·평가 비유
- "유일한 자연스러운 선택" → "적합한 선택"
- "결정적인 ~" → "중요한"
- "정공법이다" → "기본 방법이다"

### E. 메타·요약체 — 사용자 책에 없음
- "한 문장으로 정리하면" → 빼거나 "즉"
- "한 줄로 줄이면 / 한 줄로 답할 수 있어야"
- "결론을 먼저 정리하면"
- "이 사실이 ~의 본질이고"

### F. 시간 강조·시간 거래
- "30분의 추가 시간이 다음 30시간의 학습을 안정시킨다" → 빼거나 풀어쓰기
- "15분 안에", "5분 안에" 강조성 시간

### G. 추상 영어 명사
- "통제 표면 / 통신 표면" → "통제 범위 / 통신 부분"
- "골격을 잡는다" → "구조를 잡는다"
- "본질" — 반복되면 LLM 정리체

### H. 격려·권유체 과잉
- "시간을 들여 정확하게 읽기를 권합니다" → "차근차근 읽으면 좋습니다" 또는 빼기
- "한 번 더 읽기를 권합니다" — 한 번 정도면 OK, 반복은 어색
- "투자 가치가 있다 / 한 번의 투자"

### I. 환영·결의 멘트
- "다음 페이지에서 만난다" → 모두 빼기
- "손을 코드 위에 올립니다" → "코드를 작성합니다"
- "다음 챕터에서 만난다" → 빼기

### J. 사물·개념의 영어식 가치 비유 (2026-05-27 추가)
- "책의 수명을 2~3년 확보한다" → "차세대 환경이 나온 뒤에도 이 책을 이어서 활용할 수 있도록 준비한다"
  - 한국에서는 책에 "수명"이라는 표현을 쓰지 않음. "확보한다"도 영어 "secure"의 직역.
- "차별화 장이다 / 차별화 1순위 장이다" → "가장 중요한 장 중 하나입니다"
- "이 책의 운영 현실을 정리하는 장입니다" → "운영 현장에서 필요한 사항을 정리합니다"

### K. 한국 행정·SI 도메인 용어 부적합 영어식 (2026-05-27 추가)
- "격식 회신 초안" → "공문 형식의 답변 초안" 또는 "행정 답변 초안"
  - "격식"은 한국 행정 용어 아님. 영어 "formal reply" 직역.
- "양면 RAG 챗봇 / 양면 ~" → 풀어 쓰기 ("시민에게는 ~을, 민원관에게는 ~을 만들어 줍니다")
  - "양면"은 영어 "two-sided" 직역. 한국에서는 두 가지를 풀어 씀.
- "격식체"는 한국 사람이 자연스럽게 쓰지만 "공문 격식체" 같은 합성은 가급적 피하기. ch03·ch04에서 추가 검토.

### L. 자기 평가·차별화 강조 — 한국·일본 책에서 안 씀 (2026-05-27 추가)
- **★ 표시 / "핵심 차별화 장" / "차별화 가치"** — 모두 금지.
  - 미국 IT 책의 "Why this chapter matters" 패턴. 한국·일본 책은 모든 장을 동등하게 다루고 독자가 알아서 선택하게 함.
  - 본문에서 자기 평가·자기 정의는 안 함. "차별화 1순위 장입니다 / 가장 중요한 장입니다" 같은 표현 모두 제거.
- 학습 동선 도식의 ★ 표시도 제거.
- 우선 읽기 권장은 가능하지만 "이 장들부터 먼저 읽으면 도움이 됩니다" 정도로. ★ 마킹·"차별화" 단어 모두 사용 안 함.
- 마케팅·홍보는 표지·서평·출판사 보도자료의 영역. 본문에서 자기 마케팅 안 함.

### M. 본문 안 자기 마케팅 — 한국 책에서 안 씀 (2026-05-27 추가)
- **"이 책이 시중의 다른 ~ 서적과 가장 크게 다른 부분입니다"** — 모두 제거.
- **"본 서적의 가장 큰 차별화"**, **"이 책의 출발점은 ~이며, 이 책의 결정적 차이입니다"** — 모두 제거.
  - 미국 책 패턴 ("What sets this book apart"). 한국 책은 본문에서 자기 책을 강조하지 않음.
  - 책 제목·표지·서평으로 들어온 독자에게 본문 안에서 또 마케팅하는 것은 어색함.
- 본 메시지를 전달하고 싶으면 사실로만 진술: "이 책은 다섯 가지 제약(망분리·온프레미스·감사·근거인용·비용)을 다룹니다." 비교·과장 없이.

### N. 모호한 한국 행정·SI 용어 (2026-05-27 추가)
- **"시민 정보"** → "민원인의 개인정보"
  - 무엇이 시민의 것인지(개인정보? 민원 내용?) 명확하지 않음.
  - 한국 개인정보보호위 규정 용어 따라 "민원인 개인정보" 또는 "시민의 개인정보".
- 추상적·모호한 한국어 + 외래어 그대로 두기 금지. 한국 행정 표준 용어로 풀어 쓰기.

### O. 공저 노트 사용 규칙 (2026-05-27 추가)
- 공저 노트는 **다른 공저자에게 알릴 협업 메모만** 적는다.
- 도식 명세 동반 작성, 활용 후보, WARN 무시 사유, 후속 작업 메모, 검토 요청 등.
- 작업 이력("한국식 톤 재작성 적용", "X절 수정") 같은 메타는 git log·진행 메모리에 두고 본문 옆에 안 적는다.

### P. 학습 목표 마무리 단락 변환 (2026-05-27 추가)

학습 목표 글머리표 뒤 마무리 단락은 영어식 비유의 직역체가 만연한 자리이다. 사용자 예시 톤을 기준으로 변환한다.

| 영어식 (직역, 사용 금지) | 한국식 (사용) |
|------|------|
| ~가 완성되면 / ~가 잡히면 | ~를 학습하면 / ~를 익히면 |
| 모든 후속 챕터가 그 위에서 동작한다 | 이 책의 나머지 장들을 쉽게 공부할 수 있다 |
| 이 장이 만드는 것은 ~이 아니라 ~이다 | 이 장에서 만드는 ~는 ~의 기본 토대가 된다 |
| 받쳐 줄 골격 / 받쳐 줄 시드 | 받쳐 줄 수 있는 기본 토대 |
| 30시간의 학습 | 다음 학습 |
| 멈추지 않도록 받쳐 줄 | 멈추지 않도록 받쳐 줄 수 있는 (자연스러운 한국식) |
| 시드를 단단히 박는 | 기본 토대를 잘 만들어 두는 |
| 다음 30시간의 학습이 멈추지 않도록 | 다음 학습이 멈추지 않도록 |

**핵심 원칙**: 마무리 단락 자체를 삭제하지 말고, 정보를 살리되 한국 IT 책의 자연스러운 문장 형태로 변환한다. 사용자 예시:

> 이 네 가지를 학습하면 이 책의 나머지 장들을 쉽게 공부할 수 있습니다. 이 장에서 만드는 코드는 다음 학습이 멈추지 않도록 받쳐 줄 수 있는 기본 토대가 됩니다.

자동 변환 도구로는 잡기 어려운 케이스. 학습 목표 마무리 단락은 챕터마다 수동 변환 필요.
