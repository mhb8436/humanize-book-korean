6. 클래스와 객체
클래스와 객체의 기본적인 개념에 대해서는 5장에서 배웠다.  객체는 상태와 행위를 가지고 있으며 클래스는 객체를 만들어내는 설계도 역할을 하였다. 클래스를 이용해서 객체를 계속 만들어 낼 수 있었다. 이번 장에서는 클래스의 자세한 내용과 구성 요소인 필드와 메서드 그리고 객체를 생성하고 사용하는 방법과  여러가지 클래스의 종류에 대해서 배워볼 것이다.

6.1		클래스의 기본
클래스는 명사형의 상태정보인 필드와 동사형의 행위정보인 메서드를 가지고 있다. 클래스를 선언하고 메서드를 선언하는 것은 앞의 예제에서 살펴봤기 때문에 지금 쯤은 눈에 익숙할 것이다. 지금부터 필드와 메서드로 이루어진 클래스를 정의하고 사용하는 방법에 대해  자세히 공부해보자

6.1.1		클래스 선언

클래스에 대한 선언은 class 클래스명 과 같이 하고 중괄호({ }, braces)를 이용하여 클래스의 몸체를 정의한다. 클래스의 몹체 안에 필드와 메서드를 정의하면 된다.  클래스의 몸체에는 보통 세 가지 구성요소가 포함된다. 클래스의 생명주기에 관련된 생성자와  클래스 또는 객체의 상태 정보를 가지고 있는 필드와 클래스와 객체의 행위를 구현한 메서드가 있다. 메서드와 필드를 선언하는 방법은 이전의 예제에서 살펴보았지만 위의 예제에서 조금 생소한 부분이 있을 것이다. 바로 생성자이다. 생성자는 객체의 생성될 때 처리해야할 내용들을 담고 있다. 생성자는 이번 장에서 상세히 다룰 것이다. 클래스를 기본적으로 구성하기 위해서는 다음의 3가지를 지켜야 한다.

class Notebook extends Computer{ } 와 같이 extends 키워드를 사용하면 extends 키워드  뒤에 있는 클래스를 상속받는다는 의미이다.  즉 extends 우측에 있는 클래스가 부모 클래스가 된다. extends 키워드의 좌측의 클래스는 당연히 자식 클래스가 된다.
인터페이스를 구현하는 것도 마찬가지이다.   class Button implements Power{ } 와 같이 implements 키워드를 이용해서 implments  좌측에 있는 클래스는 implments 키워드 우측에 있는 인터페이스를 구현한다는 의미이다. 인터페이스를 구현할 때에는 interface 내에서 선언된 메서드를 반드시 구현해야 한다.
부모클래스를 상속받은 자식클래스나 인터페이스를 구현한 클래스는 모두 생성자와 메서드 변수를 가질 수 있다.   상속이나 인터페이스 구현을 위해서 다음의 규칙을 지켜야 한다.

<< 블럭 시작>>
extends 키워드 우측에는 상속 받을 부모 클래스를 지정해야 하고 좌측에는 부모클래스를 상속 받을 자식 클래스를 지정하면 된다.

implements 키워드 우측에는 인터페이스를 지정해야 하고, 좌측에는 인터페이스를 구현할 클래스를 지정하면 된다.
<<블럭 끝>>

예제 6.2에서 extends 키워드를 사용해서 6.1에서 만든 Computer 클래스를 상속하였다. Notebook 클래스에서는 Computer 클래스에서 정의된 필드와 메서드를 그대로 사용할 수 있다. 예제 6.2에서는 자식 클래스에서만 사용할 수 있는 int display; 와 int keyboard; 필드를 선언하였다.  Notebook 클래스의 생성자를 자세히 살펴보면 super(processor, memory)와 같은 구문이 있다. super 키워드는 부모 객체를 호출 할 때 사용하는 키워드이다. 이는 뒤에서 자세히 다룰 것이다.
그럼 인터페이스를 구현하는 클래스도 같이 공부하여보자. 예제 6.3은 인터페이스와 이를 구현한 클래스이다.

[예제 6.3] 인터페이스와 구현체
// 인터페이스 선언, 파일명은 Power.java
public interface Power {
public void setPowerOn();
public void setPowerOff();
}


// 인터페이스 구현체, 파일명은 Button.java
public class Button implements Power {
boolean isOn;

public void setPowerOn() {
isOn = true;
}

@Override
public void setPowerOff() {
isOn = false;
}

public static void main(String[] args){
Button myButton = new Button();
myButton.setPowerOn();
System.out.println("전원은 " + myButton.isOn);
myButton.setPowerOff();
System.out.println("전원은 " + myButton.isOn);
}
}

---- 출력 결과 ----

전원은 false

Button 클래스에서는 인터페이스에서 선언한 두 개의 메서드 setPowerOn()과 setPowerOff()의 세부내용을 구현하였다. 예제에서는 아주 간단하게 isOn 필드에 true 또는 false 를 할당 함으로써 전원의 on off 유무를 출력할 수 있도록 하였다.
Button 클래스의 main 메서드에서는 Button 객체를 생성하고 그 객체의 setPowerOn(), setPowerOff() 메서드를 호출하여 isOn 필드의 상태가 변하는 것을 출력하였다.

6.1.2		변수 선언
지금까지 클래스의 선언 방법에 대해 살펴봤다. 클래스의 몸체에는 필드와 메서드를 구현할 수 있다. 이제 부터는 클래스는 상태 정보를 담고 있는 필드에 대해 살펴보자.  자바에서 변수는 크게 세가지 종류가 있다.

<<블록 시작>>
1. 메서드 내부에 선언하는 변수이다. 이 변수는 로컬 변수라고 부른다.
2.클래스의 멤버 변수이다.  이 변수는 클래스 내, 메서드 외부에 선언하는 변수를 필드라고 부른다.
3. 메서드 선언 시에 포함되는 변수이다. 예를 들어  void setMemory(int memory){} 와 같이 메서드를 선언할 때 int memory와 같은 변수이다.

<<블록 끝>>

메서드 내에서 사용되는 로컬 변수는  다음 메서드를 다룰 때 자세하게 공부할 것이다. 메서드를 다룰 때에 메서드 선언 에 포함되는 변수도 같이 배울 것이다.  이번 섹션에서는 클래스의 멤버 변수에 대해 자세히 공부해보자.

우리는 클래스 내에 멤버 변수가 사용된 경우를 앞서 예제에서 살펴보았다. 클래스의 멤버 변수는 다음과 같이  선언할 수 있다.

<<블록 시작>>
선언방법)
접근제어자 변수타입 변수명;
접근제어자 변수타입 변수명 = 값;

사용예)
public class Computer{
public int processor; // 멤버 변수 선언
private int memory = 0; // 멤버 변수 선언, 선언과 동시에 할당
int disk; // 멤버 변수 선언
}

멤버 변수는 선언과 동시에 값을 할당 할 수도 있다. 이 값은 메서드 내부에서나 생성자에서 변경이 가능하다. 멤버 변수를 선언할 때 클래스와 마찬가지로 접근제어자는 반드시 지정해야 하는 요소는 아니다.
필드에서 접근제어자는 필드에 접근하는 범위를 지정한다. public 이라고 지정하면 이 필드가 포함된 애플리케이션 내부의 모든 객체에서 해당 필드에 접근이 가능함을 의미하고 private라고 지정하면 필드가 지정된 클래스 밖에서는 절대 사용이 불가능하다. protected의 범위는 패키지 내부에서만 참조된다.  필드의 경우 public 선언하여 사용해도 되지만 가능하면  private로 지정해서 사용하는 것이 앞서 배운 객체지향 프로그래밍의 캡슐화 개념에 적합하다. private로 지정한 필드는 해당 클래스의 메서드에서만 접근이 가능하고 이 메서드를 통해서 필드의 상태를 반환하거나 설정하는 것이 좋다. 즉 예제 6.4 과 같이 public 으로 지정된 메서드를 이용하여 private로 선언된 필드를 캡슐화하는 것이 바람직한 방법이다.

[예제 6.4] private 필드의 선언과 public 메서드

public class Computer {
private int cpu;
private int memory;
private int harddisk;

public Computer(int cpu, int memory, int harddisk) {
this.cpu = cpu;
this.memory = memory;
this.harddisk = harddisk;
}
public int getCpu() {
return cpu;
}
public void setCpu(int cpu) {
this.cpu = cpu;
}
public int getMemory() {
return memory;
}
public void setMemory(int memory) {
this.memory = memory;
}
public int getHarddisk() {
return harddisk;
}
public void setHarddisk(int harddisk) {
this.harddisk = harddisk;
}
}

예제 6.4에서 필드는 모두 private로 지정되어 있다. 필드를 외부에 제공하는 메서드들은 모두 public 으로 지정되어 있다.

접근제어자 다음에는 변수 타입이 지정되어 있다. 여기서는 기본 자료형인 int가 선언되어 있다. 아직 우리가 기본 자료형 밖에 배우지 않았기 때문에 예를 기본 자료형으로 들었지만 String, Object와 같은 객체 자료형도 올 수 있다. 심지어 우리가 만든 객체도 이 변수 타입에 올 수 있다.  객체 자료형은 뒤에서 자세히 배울 것이다.

변수 타입 다음에는 변수 명이 지정되었다. 자바에서 모든 변수는 동일한 변수명 명명 규칙을 따른다. 변수명에는 유니코드 문자와 숫자를 포함할 수 있고 대소문자를 구분한다. 또한 ‘$’기호나 ‘_’ 기호도 사용할 수 있지만 이 기호로 변수명을 시작할 수 없다. 또한 예약어도 사용할 수 없으며 한 단어만 사용하는 경우에는 소문자도 상관없지만 두 개의 단어를 조합할 경우에는 뒤의 단어의 첫문자는 대문자로 하는 것이 좋다. 자세한 내용은 2장 에서 공부하였다.


6.1.3		메서드 정의
메서드를 선언하는 구문은 앞서 공부한 예제에서 많이 봐서 익숙할 것이다. 특히 우리는 2장과 3장에서 예제를 공부할 때 public static void main(String[] args) 메서드 내에 예제 소스코드를 넣어서 사용하였다. 이번 섹션에서는 메서드를 선언하는 방법과 명명규칙, 그리고 메서드 오버로딩(Overloading)에 대해 배울 것이다. 우선 메서드를 정의하는 방법부터 살펴보자.

접근제어자 반환타입 메서드명( 변수타입 변수명, 변수타입 변수명 … ) {

return 반환타입에 맞는 변수
}

사용예)

}

메서드를 선언할 때 반드시 필요한 필수 구성요소들은 메서드의 반환타입, 메서드 명, 소괄호( (,) ) 그리고 메서드 몸체를 나타내는 중괄호( {, } )이다.
소괄호 내에는 보통 메서드에 넣을 매개변수의 자료타입과 매개변수명이 들어간다. “,”을 기준으로 여러 개가 들어갈 수 있다.
반환타입은 기본자료형, 객체자료형 모두 가능하며 반환타입이 없는 경우에도 반드시 void 로 반환타입을 기재하여 반환타입이 없음을 명시적으로 컴파일러에게 알려주어야 한다. 반환타입이 있는 경우에는 메서드 몸체 내부에서 return “반환타입과 동일한 자료형을 가진변수” 과 같이 지정해주어야 한다. 그렇지 않으면 컴파일 에러를 발생 시킨다. 예제를 통해서 메서드를 만드는 방법을 알아보자. 예제 6.5는 여러가지 메서드의 종류와 사용법이다.

[예제 6.5] 메서드의 정의와 사용법
public class DeclareMethodExample {
// 반환타입 : void, 매개변수 : int a

System.out.println(a);

일반적인 메서드의 구성요소

*접근제어자 : public, private와 같이 메서드에 접근 범위를 설정할 수 있다. 필드와 마찬가지로 public은 해당 메서드를 포함한 애플리케이션 어디에서나 접근할 수 있고, private는 메서드가 포함된 클래스에서만 접근이 가능하다.
*반환타입 : 메서드에 의해서 반활될 데이터 타입이다. 반활될 데이터가 없으면 void로 지정하면 된다.
*메서드명 : 필드의 명명 규칙은 메서드에도 적용될 수 있고, 메서드의 경우에는 한 단어인 경우에는 소문자이고 두개 이상의 단어가 연결되면 두번째 단어부터 첫글자는 대문자가 오는 것이 좋다.

*메서드 몸체 : 메서드의 몸체는 반드시 중괄호 ( {} )로 감싸야 한다. 중괄호 내에 있는 코드만이 메서드의 코드로 인식된다.

6.1.4		생성자
클래스는 객체를 생성할 때 호출하는 생성자를 기본적으로 가지고 있다. 생성자는 반환 타입이 없고 클래스의 이름을 그대로 이용한다. 생성자에서 주로 처리하는 로직은 객체를 생성할 때 필요한 사항들을 담고 있다. 예를 들어 클래스에서 필요한 변수를 외부에서 초기화 할 경우가 있다. 다음의 생상자 코드 조각을 살펴보자.

class Notebook{

int memory;
int hardDisk;
public Notebook(int processor, int memory, int hardDisk){
this.processor = processor;
this.memory = memory;
this.hardDsik = hardDisk;

생성자의 모습은 메서드와 아주 흡사하다. 단지 void, int 와 같은 반환 타입이 없고 클래스 명과 동일하다. 생성자는 메서드와 마찬가지로 매개변수를 받을 수 있다. 위의 코드 조각에서는 생성자의 매개변수를 필드에 할당 하였다.  생성자는 주로 필드의 값을 외부에서 초기화 하기 위한 목적으로 많이 사용된다. 생성자를 이용해서 외부에서 생성하는 코드 조각을 살펴보자.

6.1.5		메서드나 생성자에 값 전달 방법
메서드나 생성자를 정의할 때 대부분의 경우에는 메서드나 생성자에 매개변수를 전달한다.  메서드나 생성자를 외부에서 호출하거나 생성할 시점에 메서드나 생성자에서 정의된 매개변수의 갯수나, 타입 그리고 순서에 맞게 값을 전달해야 한다.  다음 메서드 예제 코드를 살펴보자.

public double sumDistinct(int a, int b, String name){
double c = Math.sqrt( a^2 + b^2 );
System.out.println(name + " 은 : " + c);
return c;
}

위의 코드 조각은 피타고라스 정리를 계산하는 간단한 코드이다. Math.sqrt의 역할은 수학 기호 √ 역할과 같다.  sumDistinct 메서드는 총 3개의 매개변수를 받아서 앞의 int 형 매개변수 두 개 a, b는 피타고라스 정리의 빗변을 제외한 두 개의 변의 길이이고 c는 빗변의 길이다. 빗변의 길이 c를 계산한 뒤에 이를  출력하고 반환하였다.

메서드나 생성자의 매개변수에는 모든 종류의 자료형이 올 수 있다. 기본 자료형 뿐만 아니라 객체나 배열과 같은 참조 자료형도 올 수 있다.  매개변수의 이름은 메서드 내에서는 유일해야 한다. 매개변수에 선언된 변수명은 메서드 내부에서 동일한 이름으로 새로 생성할 수 없다는 의미이다. 예를 들어 다음과 같은 코드는 컴파일 시에 에러를 발생하게 된다.

public int sumValues(int a, int b){
int a = 0; // a라는 이름은 매개변수 a와 중복되기 때문에 컴파일 에러 발생
// 기타 메서드 내용
}

하지만 클래스에서 사용된 필드의 이름과 메서드의 매개변수의 이름은 같아도 상관이 없다. 하지만 메서드 내부에서는 매개변수의 이름이 우선하게 된다. 예를 들어 클래스의 필드에 int a 라는 변수가 선언이 되어있고 메서드의 매개변수에 int a 가 동일한 이름으로 있을 경우에도 컴파일 에러는 발생하지 않지만 메서드 내부에서 a를 사용하려고 할 때에는 매개변수의 변수 a가 참조된다. 즉 필드의 a 는 그 메서드 내부에서는 a라는 이름으로 참조할 수 없다. 필드 a를 참조하기 위해서는 this.a 와 같이 this 키워드를 사용해야 한다. this 키워드는 뒤에서 다시 자세하게 다룰 것이다.

메서드의 매개변수의 타입이 기본형 타입, 참조형 타입에 따라서 인자로 할당된 값을 메서드 내부에서 변경을 했을 때 메서드를 호출한 곳에서 봤을 때 그 값이 변경이 될까? 결론적을 우선 말씀드리자면, 매개변수가 int, double, char와 같은 기본 자료형일 경우에는 메서드 내부에서 매개변수에 무엇을 할당 하더라도 메서드를 호출한 곳에서는 값이 변하지 않는다. 백문이 불여일타! 이해를 돕기 위해 실제 예제를 만들어 보자.

예제 6.7에서 출력 결과는 무엇일까? 정답은 3이다.  기본 자료형의 매개변수를 가진 메서드에 인자를 할당할 경우에는 매개변수의 가용 범위는 그 메서드 내부이다. 참조 자료형인 경우에도 인자로 전달한 객체를 메서드 내부에서 바꾸었다고 하더라도 메서드 내부에서만 유효하고 메서드를 호출한 곳에서는 여전히 메서드를 호출하기 전의 객체를 참조하고 있다. 예제를 통해서 알아보도록 하자.

6.2		객체의 기본
자바 API는 굉장히 많은 객체를 제공한다. 이 객체를 통해서 GUI(Graphic User Interface)도 구현을 하고, 네트워크를 통해서 통신도 하고, 서버도 실행시키고, 애니메이션도 구동한다. 또한 객체는 임무가 종료되더라도 나중에 재 사용될 수 있다.  객체는 생성에서 소멸할 때까지 생명주기를 가지고 있다. 객체의 생성부터 차례로 학습해보자.


6.2.1		객체 생성방법
클래스는 객체의 설계도이고, 객체는 클래스에서 만들어진다. 객체를 생성하는 구문은 다음과 같이 생성할 수 있다.

String name = new String();
Computer myComputer = new Computer(name, 2, 4, 500);
Computer myComputer = new Computer(name, 4, 8);

String name = new String()에서는 new 키워드를 이용해서 새로운 String 클래스의 객체를 만들었다. 그 객체의 이름은 name이다. 두 번째 줄의  Computer myComputer = new Computer(name, 2, 4, 500); 에서는 String 객체와 기본 자료형을 인자로 넘겼다.  두 번째 줄의 객체를 생성하는 구문을 자세히 살펴보자.  첫 번째 Computer myComputer 처럼 myComputer 라는 이름으로 Computer 타입의 변수를 하나 선언하였다. 이는 2장의 자료형에서 변수를 선언할 때 배웠던 구문이다. 두번째 = 기호의 우측편에 new Computer( 와 같이 Computer  객체를 생성하였다. 세번째 Computer(name, 2, 4, 500)과 같이 생성자에 인자를 넘겨서 객체를 초기화 하였다. 네 번째 생성되고 초기화된 객체를 = 기호 왼편의 myComputer 변수에 할당하였다.

<<블록 시작>>
1. 선언 : 객체 타입의 변수를 선언한다.  예) Computer myComputer
2. 생성 :  new 연산자를 이용해서 객체를 생성한다.  예) new Computer

객체를 선언만 한 상태에서는 객체를 사용할 수 없을 명심하자. 객체를 사용하기 위해서는 클래스를 인스턴스화 해야 한다.

<<블록 시작>>

인스턴스화(instantiation) 이란 말은 클래스를 객체로 만든다는 의미이다.  인스턴스란 단어의 사전적 의미는 ‘즉석’, ‘즉시’ 라는 의미가 있다. 인스턴스화는 즉석화 하다라는 의미 정도가 될 것 같다. 자바 프로그래밍을 공부하거나 영어로 된 문서를 보는 경우에 클래스를 인스턴스화 하다라는 단어가 자주 등장하는데, 클래스에서 객체를 만든다. 또는 클래스를 이용해서 객체를 즉석에서 찍어낸다라는 개념으로 이해하면 될 것 같다. 또한 클래스의 인스턴스는 객체와 같은 의미가 될 것이다.
<<블록 끝>>

new 연산자를 이용해서 하나의 객체만 생성 할 수 있다. 즉 new 연산자를 이용해서 여러 개의 객체를 동시에 생성할 수 없다. new 연산자를 이용해서 객체를 만들게 되면 신규 객체를 위해 메모리가 할당되고 메모리의 주소가 반환된다. 그리고 나서  new 연산자는 객체의 생성자를 호출하게 된다.

Computer myComputer = new Computer(name, 2,4)

위의 코드에서 new 연산자를 이용해서 Computer 객체를 생성하고  생성자를 이용해서 객체를 초기화 한 뒤에 new 연산자는 객체의 주소를 반환한다. 반환된 주소는 myComputer 변수에 할당이 된다. new 연산자에서 반환된 값을 구지 myComputer 라는 이름의 변수에 핧당하지 않고 사용할 수도 있다.

(new Computer(name,2, 4)).getMemory();

위의 코드와 같이 호출하면 생성된 객체를 별도의 변수에 할당 하지 않고도 참조해서 사용할 수 있다. 하지만 내부적으로는 임의의 변수가 할당이 된다.

우리는 앞서 객체를 초기화 하기 위한 생성자를 학습하였고 생성자를 만드는 방법과 사용하는 방법도 눈에 익숙할 것이다. 지금부터는 생성자를 이용한 객체의 초기화 과정을 살펴보자.  백문이 불여일타! 다음 예제를 6.9의 소스코드를 직접 타이핑하고 실행해보자.

[그림 6.1] 객체 생성 프로세스


그림 6.6은 예제 6.9의 객체 생성 프로세스를 그림으로 표시한 것이다. String 클래스는 자바에서 제공하는 문자열을 다룰 수 있는 클래스이다. ① 에서 String 클래스 타입의 변수 name을 선언하고 ② 에서 String 클래스의 생성자를 이용해서 “Desktop Computer”라는 문자열을 가진 새로운 객체를 생성하고 나서 ③ new 키워드에 의해 반환된 생성된 객체의 주소값을 선언된 변수에 할당하였다.
두 번째로 ④ DesktopComputer d1 으로 DesktopComputer 객체 타입을 선언하였다. new 키워드로 ②와 동일하게 ⑤ 에서 객체를 생성하였고 생성자를 이용해서 앞서 생성한 String 객체와 기본형 인자를 생성자의 매개변수로 넘겼다. 이렇게 생성된 객체의 주소는 반환 되어 d1 변수에 주소값이 할당되었다.  여기서는 매개 변수가 있는 생성자를 이용해여 객체를 생성하였지만 생성자를 클래스에 만들지 않아도 객체의 생성이 가능하다. 자바 컴파일러는 생성자가 없는 클래스의 경우에는 매개변수가 없는 생성자를 만들어준다. 매개변수가 없는 생성자를 우리는 기본 생성자(default constructor)라고 부른다.


6.2.2		객체 사용방법
지금까지 객체를 생성하는 방법을 배웠다. 객체를 생성할 때에는 객체를 사용할 것을 예정을 하고 생성하였을 것이다. 예를 들어 메서드를 이용하여 로직을 처리하거나 필드를 이용하기 위한 목적일 것이다.

객체는 클래스를 기반으로 생성되었기 때문에, 생성된 객체의 필드는 모두 별도의 공간에 저장된다. 따라서

Computer d1 = new Computer();
Computer d2 = new Computer();

d1의 cpu 필드에서 사용하는 공간과 d2의 cpu 에서 사용하는 저장공간은 모두 다르다.  각 객체의 cpu 필드에 접근하기 위해서는 d1.cpu, d2.cpu와 같이 생성된 객체명 뒤에 점( . ) 기호와 필드명을 이용하여 필드에 접근한다. d1.cpu와 d2.cpu는 모두 다른 공간을 바라 본다는 사실을 명심하자.

반드시 객체 타입의 변수를 선언한 뒤에 그 객체의 필드를 사용할 있는 것만은 아니다. 변수를 선언하지 않고도 객체의 필드를 참조할 수 있는 방법이 있다. 객체를 생성과 동시에 필드를 참조할 수도 있다.

int cpu= new Computer().cpu;

와 같이 new Computer() 생성자를 이용해서 객체를 생성함과 동시에 그  객체의 cpu 필드에 접근할 수 있다. 생성자에 아무 인자도 넣지 않았기 때문에 cpu 에는 기본값(default value)만 존재하고 이 값을 반환할 것이다. 하지만 이렇게 하게 되면 다음 구문들에서 이 구문에서 생성된 객체에 다시 접근할 방법이 없다. 생성된 객체를 재사용하기 위해서는 객체 타입의 변수를 선언하고 생성된 객체를 할당 한 뒤에 사용해야 한다.

생성된 객체에서 메서드를 사용하는 방법도 필드와 동일하다. 생성된객체명과 점( . ) 기호와 메서드명을 이용하면 쉽게 메서드를 참조할 수 있다. 메서드 참조 시 객체와 다른 것은 인자를 넣을 수 있는 괄호를 사용해야 한다는 점이다. 인자가 없다고 하더라도 다음과 같이 비어있는 상태로 괄호를 열고 닫아야 한다.

<<블록 시작>>
메서드 호출 방법

생성된객체명.메서드명(인자목록);
생성된객체명.메서드명();

사용예)

Computer myComputer = new Computer();
myComputer.getCpu();
myComputer.setCpu(2);

<<블록 끝>>

객체의 메서드를 호출 할 때 넘겨줄 인자가 없더라도 반드시 괄호를 열고 닫아야 한다는 사실을 명심하자. 메서드에서 어떤 값이 반환된다면 메서드를 호출 한 뒤에 변수에 값을 할당 할 수 있다.  예를 들어

int cpu = myComputer.getCpu();

와 같이 myComputer 객체에서 getCpu() 메서드를 호출한 뒤에 반환된 결과를 cpu 변수에 담을 수 있다. 메서드에서 반환된 결과를 별도의 변수에 담지 않고 바로 사용할 수 도 있다.

System.out.println( myComputer.getCpu() );

와 같이 myComputer.getCpu() 메서드를 호출 한 뒤에 System.out.println 메서드로 화면에 반환 결과를 출력할 수 있다.   또한 필드와 마찬가지로 객체를 성성함과 동시에 메서드를 사용할 수 있다.

int cpu = new Computer(2,4,500).getCpu()

와 같이 new 연산자로 Computer 객체를 생성하여 생성자의 인자로 객체를 초기화 한뒤에 바로 getCpu() 메서드를 호출하여 초기화된 필드의 값을 반환 받을 수 있다. 하지만 필드와 마찬가지로 다른 구문에서 생성된 객체를 재사용해야하는 경우라면 변수에 객체를 담아서 사용해야 한다.

6.3		클래스의 고급 사용법
이번 장에서는 메서드에서 return 문을 사용하여 값을 반환하는 방법과 this 키워드와 접근제어자에 대해 공부할 것이다. 또한 static 키워드를 사용해서 클래스 변수와 클래스 메서드를 선언하고 사용하는 방법을 배울 것이다.

6.3.1		메서드와 return
메서드에서 값을 반환 하는 경우는 return 구문을 만나거나 예외가 던져 지는 경우이다. 메서드에서 값을 반환 하기 위해서는 return 에서 반환하는 값의 자료형과 메서드에서 선언하는 반환 타입과 일치해야 한다. 메서드에서 값을 반환하는 경우에는 메서드 몸체 어디에선가는 return 문으로 값을 반환 해야 한다. 그렇지 않으면 컴파일 시점에 에러가 발생한다. return 구문은 if 문이나 for 문과 같은 제어 흐름 로직에서 나오거나 메서드를 종료하기 위해서 사용된다. 다음의 예제 6.11은 if 문의 결과에 따라 return문을 사용하는 예제이다.

[예제 6.11]return문의 사용하는 예제
public class ReturnExample {

public boolean checkZero(int i){
if( i == 0 ){
return true;
}
return false;
}
public static void main(String[] args) {
ReturnExample test = new ReturnExample();
System.out.println("결과는 " + test.checkZero(0));
System.out.println("결과는 " + test.checkZero(1));
}
}

값을 반환하지 않아도 void 으로 반환 타입을 선언해야 한다. 메서드에서 반환타입을 void 로 선언하게 되면 메서드 내에서 구지 return 구문을 사용하지 않아도 된다. void로 선언하고 return 구문으로 값을 반환하면 컴파일 시점에 에러를 발생하게 된다. 다음 코드는 컴파일 에러를 발생시킨다.

public void checkZero(int i){
return i; // Void methods cannot return a value 에러가 발생한다.
}

메서드의 반환 타입에는 기본 자료형 이외에도 객체 타입도 반환할 수 있다. 다음의 예제 6.12에서는 객체 형인 Computer 객체를 반환한다.

6.3.2		this
앞서 클래스 멤버 변수와 메서드의 로컬변수의 차이점에 대해 공부할 때 this 키워드에 대해 언급한 기억이 날 것이다. 멤버 변수와 로컬 변수의 명칭이 같을 경우에 메서드 내부에서 일반적인 방법으로는 멤버 변수를 사용할 수 없었다. 멤버 변수를 사용하기 위해는 this.멤버변수명 으로 사용해야 한다고 공부하였다. 지금부터 this 키워드의 의미가 무엇인지 살펴보자. this 키워드는 의미대로 자금 바로 이(this) 객체를 의미한다. 객체는 설계도인 클래스를 이용해서 찍어  낸다고 하였다. - 공장의 프레스 기계에서 하나의 틀로 동일한 제품을 계속 만들어 낸는 모습이 클래스에서 객체가 생성되는 모습과 유사하여 찍어 낸다라는 단어를 선택하였다. -  this 키워드는 찍어낸  객체 바로 그 자신을 의미한다. 다음 그림 6.7에서 이를 도식화 하였다.

위의 소스코드에서 생성자 내부 cpu = cpu; memory = memory; harddisk = harddisk; 라고 할당하였다. 이전에 우리가 공부했던 Computer 클래스는 모두 this.cpu = cpu; 와 같이 할당하였다. 위의 코드와 같이 클래스를 만들고 클래스를  생성하게 되면 클래스의 멤버 변수에는 아무런 값이 할당되지 않는다. 즉 생성자 내부의 cpu와 memory, harddisk는 모두 로컬 변수로 인식이 된다. 위의 코드의 Computer 클래스를 이용해서 객체를 생성하면 클래스 멤버 변수인 cpu, memory, harddisk 에는 항상 int 자료형의 기본 값인 0만 들어가 있을 것이다.  위의 코드를  제대로된 코드로 바꾸어 테스트를 진행해보자. 예제 6.13은 정상적인 소스 코드 이다.

6.3.3	접근제어자
public, private와 같은 접근제어자는 클래스나 메서드 또는 필드에 접근할 수 있는 범위를 지정한다. 접근 제어자를 설정하지 않는 경우도 봤을 것이다. 접근제어자를 지정하지 않으면 객체, 메스드, 필드에는 접근할 수 있는 범위가 다른 접근 제어자와는 달라진다.

<<블록 시작>>
* class의 접근 제어자  :  public,  접근제어자를 지정하지 않음
* 멤버 변수/메서드의 접근 제어자 : public, private, protected, 접근제어자를 지정하지 않음
<<블록 끝>>

클래스 선언 시에 붙이는 접근 제어자는 public과 접근제어자를 지정하지 않는 경우가 있었다. public의 경우에는 모든 클래스, 패키지에서 해당 객체에 접근이 가능함을 의미한다. 접근제어자를 지정하지 않는 클래스는 해당 패키지와 하위 패키지에서 객체에 접근할 수 있다.

멤버 변수나 메서드의 접근 제어자는 public, private, protected 그리고 접근제어자를 지정하지 않는 경우 총 4가지가 있다. public은 모든 클래스에서 접근이 가능하고 접근제어자를 지정하지 않는 경우는 클래스가 포함된 패키지에서만 접근이 가능하다. private 접근제어자를 지정하는 필드와 메서드는 해당 클래스에서만 참조할 수 있다. 즉 외부의 클래스에서는 이 접근제어자로 지정된 필드나 메서드에는 접근할 수 없다. protected 접근 제어자는 해당 패키지에서 접근이 가능하다. 필드와 메서드에서 접근제어자를 지정하지 않는 경우에는 특이하게도 상속받은 클래스에서도 접근이 가능하다.

[표 6.1] 접근 제어자의 접근 가능 범위

표6.1은 접근제어자의 지정범위를 나타낸다. public 접근제어자의 경우에는 해당 class, 클래스의 패키지 package, 하위 클래스, 전역적으로 접근이 가능하다. 접근제어자를 지정하지 않는 경우에는 해당 class, 클래스의 패키지 package 에서만 접근 가능하다. protected를 지정하게 되면 해당 class, 클래스의 패키지, 하위 클래스에서 접근이 가능하다.  private 접근제어자는 해당 클래스에서만 접근이 가능하다. 접근제어자를 지정하지 않는 경우와 protected의 접근 범위를 명확히 구분해야 한다. protected 접근제어자는 상속된 자식 클래스에서 접근이 가능하지만 접근제어자를 지정하지 않는 경우에는 자식 클래스에서는 접근이 불가능하다.


6.3.4	클래스 변수와 클래스 메서드
지금까지 우리는 객체의 변수와 객체의 메서드에 대하여 공부하였다. 객체란 클래스를 이용해서 만들어진 인스턴스이다. 즉 객체 내의 메서드와 변수는 그 객체에서만 유효하다. 그럼 클래스를 이용해서 만들어진 모든 객체에 적용할 수 있는 메서드와 변수를 만들 수 있을까?  자바에서는 클래스에 적용하여 모든 메서드와 변수에 적용할 수 있는 방법을 제공한다. 멤버 변수나 멤버 메서드에 static 키워드를 사용하면 그 멤버 변수나 메서드는 모든 객체에 적용이 될 수 있다.  static 키워드가 적용된 필드와 메서드를 클래스 변수, 클래스 메서드라고 부른다.

클래스 변수는 모든 객체에서 호출해도 동일한 값을 가지는 클래스의 변수이다. 클래스로부터 만들어진 모든 객체에서 클래스 변수로 설정된 값을 사용할 수 있다. 만일 다른 객체에서 그 값을 변경하게 되면 모든 객체에 동일하게 적용이 된다. 클래스 변수는 객체를 생성하는 것과 같이 어떠한 생성 로직 없이 사용될 수 있다. 클래스 변수는 개별 객체의 생명주기에 상관 없이 상태를 저장하고 관리하고 싶을 때 사용한다. 백문이 불여일타! 다음의 예제코드를 만들어 보자.

[예제 6.15] static을 이용한 클래스 변수

예제 6.15와 같이 3개의 객체를 생성 한 뒤에 Computer.numberofComputer변수를 출력하면 3이란 값이 출력된다. 즉 numberofComputer++ 구문은 3번 호출 되었고 우리는 위에서 3개의 객체를 생성하였다.  객체가 매번 생성되어도 numberofComputer 변수는 변함없이 값을 증가시켰다. 즉 클래스 변수는 객체의 생성과는 상관없이 클래스의 상태를 유지할 수 있다.

클래스 메서드도 클래스 변수와 마찬가지로 사용되고 정의될 수 있다. static  키워드를 메서드 선언시에 사용할 수 있고 클래스명.메서드명(인자) 의 형태로 사용할 수 있다. 클래스 메서드를 사용하는 방법을 예제를 통해서 알아보자.

즉 static으로 선언되지 않았다. 예제 6.16에서 getNumberofComputer 메서드를 위의 코드조각으로 바꾼 뒤에 컴파일 해보자. 컴파일 에러가 발생 할 것이다. 왜 컴파일 에러가 발생했을까?
static으로 선언하게 되면 모든 객체에 영향을 미친다고 하였다. cpu는 객체가 생성이 될 때마다 생성되는 변수이다. static으로 선언된 메서드에서 객체마다 생성되는 cpu 변수를 반환하면 어떤 객체의 cpu 를 반환해야할까? 컴파일러도 이 부분을 애매모호하게 생각하여 컴파일 에러를 발생하는 것이다. 이를 해결 하기 위해서는 cpu 변수 앞에 this 키워드를 추가하여 this.cpu로 호출한 해당 객체의 cpu 변수가 반환 된다고 명시해줘야 한다.

그럼 인스턴스 메서드에서 static이 선언된 클래스 메서드나 클래스 변수를 부를 때는 컴파일 오류가 없을까? 다행이도 객체 별로 생성되는 인스턴스 메서드에서 클래스 변수나 클래스 메서드를 부를 때는 컴파일러는 헤깔려하지 않을 것이다. 클래스 메서드에서 클래스 변수나 메서드를  호출해도 컴파일러는 혼동하지 않는다.

변수가 있다면 상수라는 개념도 존재한다. 변수가 항상 변하는 값이라면 상수는 변하지 않는 값을 의미한다. 인스턴스 메서드 내부에서 상수를 선언하여 사용할 수 있지만 일반적으로 static 키워드와 같이 사용하여 클래스 상수로 선언하여 많이 사용한다.

static final int  LOOP = 1000;

위의 코드 조각과 같이 LOOP  상수에 1000 이라는 값을 선언과 동시에 할당하게 되면 LOOP 에 저장된 1000 이란 값은 절대 변함이 없다. 상수는 주로 이렇게 변동이 되면 안되는 값을 저장할 때 자주 이용되며 보통은 상수명을 대문자로 주로 사용한다. 만약 여러 개의 단어를 사용하는 경우에는 LOOP_OF_FOR와 같이 언더바(_)를 이용하여 단어를 연결하여 사용한다.

static 키워드를 사용하지 않는 인스턴스 변수의 경우에는 생성자나 별도의 인스턴스 메서드를 통해서 변수를 초기화 하면 된다. 변수를 초기화 한다는 의미는 변수에 초기 값을 할당한다는 의미이다. 값을 직접적으로 할당 하는 경우도 있지만 어떤 조건에 따라서 초기화를 다르게 한다거나 반복문을 돌면서 초기화를 해야하는 경우도 있다. 예를 들어 배열 같은 경우 초기화를 위해서 반복문을 수행해서 초기화를 진행해야 하는 경우도 있을 것이다. 인스턴스 변수를 이렇게 초기화 해야한다면 생성자나 별도의 초기화 인스턴스 메서드에서 수행하면 된다.
예를 들어 다음 코드와 같이 초기화를 수행 하면 된다.

class Computer{
private int cpu;
public Computer(int cpu){
// 초기화 로직 수행
}
}

위의 코드 조각처럼 생성자에 초기화 로직을 추가할 수 도 있고 별도의 인스턴스 메서드에 초기화 로직을 담고 초기화를 수행할 수도 있다. 예를 들어 다음 코드와 같이 수행하면 된다.

class Computer{
private int cpu = initializeCpu();
public int initializeCpu(){
// 초기화 로직 수행
}
}

위의 코드는 initalizeCpu()메서드를 이용해서 초기화를 수행하였다. 인스턴스 변수 초기화를 위해서 또 다른 방법도 있다.

class Computer{
private int cpu;
{
// cpu 초기화 로직 수행
}
}

위의 코드와 같이 블럭 기호( { } )를 열고 닫고 블럭 기호 몸체에 초기화 로직을 수행할 수도 있다. 위의 방법들은 모두 장 단점이 있지만 초기화 로직의 재사용을 염두해 둘 경우에는 별도의 인스턴스 메서드를 통해서 초기화 로직을 수행하는 방법이 좀 더 효과적이다.

static 키워드를 사용한 클래스 변수의 초기화를 위해서서는 static  구문의 블럭을 사용하면 된다.

<<블록 시작 >>
사용 방법

static {
// static 변수의 초기화 로직 또는 값을 넣는다.
}

사용 예)

static int numberofComputer;

static {
numberofComputer = 1;
}
<<블록 끝>>

위와 같이 static {} 블록을  사용하여 변수를 초기화 할 수도 있지마 별도의 클래스 메서드를 만들고 이를 호출함으로써 초기화 할 수 있다.

class Computer{
public static String realName = initializeName();
private static String initializeName(){
// static 변수 realName을 위한 초기화 로직 삽입
}
}


6.4	중첩 클래스
자바 프로그램에서는 클래스안에 클래스를 정의할 수 있다. 이렇게 복잡한 개념을 도입한 이유는 첫 번째 논리적으로 프로그램 로직을 그룹핑 하기 위해서 이다. 예를 들어 HardDisk 라는 클래스가  있고 HardDisk 클래스에서만 사용되는  HardDiskHelper 클래스가 있다고 가정하자. HardDiskHelper 클래스는 구지 별도의 파일로 빼서 작업할 필요 없이 HardDisk 클래스의 중첩 클래스로 사용하면 된다.
두 번째  정보 은닉의 목적으로 사용될 수 있다. 예를 들어 Computer 클래스 내부에 Memory 클래스가 있다고 가정하자. Computer 클래스 외부에서 Memory 클래스의 내용을 구지 알 필요가 없다고 하자. 그럼 Memory 클래스를 Computer 클래스 내부에 넣어서 외부에서는 Memory 클래스를 참조할 수 없게 할 경우 사용한다.
중첩 클래스는 클래스 내부에 선언된 클래스이다. 중첩 클래스는 클래스 내부에 선언된 클래스를 의미한다.  중첩 클래스를 선언할 때 static 키워드를 붙이게 되면 클래스 변수와 같이 취급되고, static 키워드를 붙이지 않으면 인스턴스 변수 처럼 취급된다. 주로 다음의 코드 처럼 선언한다.

class Computer{

static class Memory{

}
class HardDisk{

|
}

static  키워드를 이용해서 클래스 변수로 선언된 중첩 클래스는 다음과 같이 접근할 수 있다.

Computer.Memory;

예를 들어 중첩 클래스를 생성하기 위해서는 다음의 구문처럼 작업하면 된다.

Computer.Memory mem1 = new Computer.Memory();

중첩 클래스는 영어로 Nested Classes라고 부른다. 중첩 클래스의 종류에는 몇 이너 클래스(Inner Class), 익명 클래스(Anonymouse Class)등이 있다.  지금부터 각각 클래스에 대해 공부해보자.

6.4.1 이너 클래스
이너클래스는 바깥 클래스의 객체 생성 시에 같이 생성되고 소멸 시에 같이 소멸되고 바깥 클래스의 인스턴스 메서드 처럼 취급되기 때문에 바깥 클래스의 인스턴스 변수와 인스턴스 메서드에 접근이 가능하다. 인스턴스 변수와 인스턴스 메서는 static 키워드가 붙지 않은 상태의 변수와 메서드이다. 이너 클래스는 바깥 클래스의 인스턴스로 사용되기 때문에 이너 클래스에서는  static 키워드를 사용할 수 없다.
이너 클래스를 초기화 하기 위해서는 바깥 클래스에서 이너 클래스를 초기화 해야 한다. 예제 코드를 살펴보면서 이너 클래스의 동작 방법을 익혀보자.

[예제 6.17] 이너 클래스 예제
public class Computer {
static int  memory;
int harddisk;

static class Memory{
int memorySize;
char memroyType;

public Memory(int memorySize, char memroyType) {
this.memorySize = memorySize;
this.memroyType = memroyType;
memory = memorySize;
}
public int getMemorySize() {
return memorySize;
}
public void setMemorySize(int memorySize) {
this.memorySize = memorySize;
}
public char getMemroyType() {
return memroyType;
}
public void setMemroyType(char memroyType) {
this.memroyType = memroyType;
}

}

class HardDisk{
int diskSize;
char diskType;
public HardDisk(int diskSize, char diskType) {
this.diskSize = diskSize;
this.diskType = diskType;
harddisk = diskSize;
}
public int getDiskSize() {
return diskSize;
}
public void setDiskSize(int diskSize) {
this.diskSize = diskSize;
}
public char getDiskType() {
return diskType;
}
public void setDiskType(char diskType) {
this.diskType = diskType;
}

}

public static void main(String[] args) {
// static 이너 클래스 생성
Computer.Memory memory = new Computer.Memory(128, 'M');
System.out.println(memory.getMemroyType() + ":" + memory.getMemorySize());
// 인스턴스 이너 클래스 생성
Computer computer1 = new Computer();
Computer.HardDisk harddisk = computer1.new HardDisk(500, 'H');
System.out.println(harddisk.getDiskType() + ":" + harddisk.getDiskSize());
// 바깥 클래스 변수 선언
System.out.println(computer1.memory + ":" + computer1.harddisk);
}
}


예제 6.17은  두 개의 이너 클래스를 가지고 있다. static이 붙어 있는 클래스와 static이 붙어 있지 않는 이너 클래스이다. static이 붙어 있는 Memory 이너 클래스는 클래스 변수 처럼 취급이 된다. Memory 클래스의 생성 방법을 살펴보자. Computer 클래스의 main 메서드에 다음의 코드를 살펴보면

Computer.Memory memory = new Computer.Memory(128, 'M');

점(.) 기호를 이용해서 Computer 클래스의 이너 클래스인 Memory 클래스에 접근하였다. new 키워드를 이용해서 Memory 클래스를 생성하였다. Computer.Memory 의 의미는 Computer 클래스 내에 있는 Memory 클래스를 의미한다라고 이해하면 될 것 같다.  두 번째로 메인 메서드 내에 있는 다음의 코드를 살펴보자.

Computer computer1 = new Computer();
Computer.HardDisk harddisk = computer1.new HardDisk(500, 'H');

위 코드에서는  Computer 객체 타입의 computer1 이라는 변수에 새로운 Computer 객체를 할당하였다. Computer 클래스 내에 있는 HardDisk 클래스는 멤버 변수 처럼 취급이 되기 때문에 Memory 클래스 처럼 Computer.Memory 와 같은 형식으로 접근할 수 없다.   HardDisk 객체를 생성하기 위해서는 computer1 변수 객체를 이용해서 computer1.new HardDisk(500, 'H') 와 같이 "변수명.new 이너클래스생성자명()" 과 같이 생성할 수 있다. 생성된 이너 클래스는 harddisk 변수를 이용해서 접근하면 된다.
예제 6.17에서 또 하나 주의 깊게 살펴볼 곳은 Memory 클래스와 HardDisk 클래스의 생성자의 마지막 라인이다. 다음의 코드들이다.

public Memory(int memorySize, char memroyType) {
this.memorySize = memorySize;
this.memroyType = memroyType;
memory = memorySize;
}

public HardDisk(int diskSize, char diskType) {
this.diskSize = diskSize;
this.diskType = diskType;
harddisk = diskSize;
}

각 생성자의 마지막 라인에 보면 memory 변수와 harddisk 변수가 있다. 이 변수들은 모두 이너 클래스에 선언되지 않고, 바깥 클래스인 Computer 클래스에 선언이 되어 있다. 이너 클래스에서는 별다른 키워드 없이 바깥 클래스의 인스턴스 변수에 접근할 수 있다. 인스턴스 메서드나 클래스 변수, 클래스 메서드에도 접근이 가능하다.


6.4.2	로컬 클래스
로컬 클래스는 클래스 내부의 블럭 내에 선언되는 클래스이다. 주로 선언되는 장소는 메서드 내부이다. 로컬 클래스는 괄호(braces,  {} )로 쌓여진 곳이면 어디나 선언이 가능핟.  예를 들어 for 루프나, if 문에서도 선언이 가능하다.  백문이 불여일타! 예제소스를 직접 만들어보면서 로컬 클래스가 어떻게 생성이 되는 지 살펴보자.

[예제 6.18] 로컬 클래스 예제

public class Member {

public static String validJuminNo(String juminNo){
final int juminLength=13;
// int juminLength=13; // final 키워드가 없어 컴파일 에러 발생
class JuminNo{
String juminNo="";
public JuminNo(String juminNo){
if(juminNo.length() == juminLength)
this.juminNo = juminNo;
else
this.juminNo = juminNo.replaceAll("-", "");
}
public String getJuminNo() {
return this.juminNo;
}
}
JuminNo jumin = new JuminNo(juminNo);
return jumin.getJuminNo();
}

public static void main(String[] args) {
System.out.println(validJuminNo("7511111234567"));
System.out.println(validJuminNo("751111-1234567"));
}
}



예제 6.18은 로컬 클래스의 예제이다. jumin 이란 단어는 주민번호를 뜻한다. 이 예제에서는 우리가 배우지 않은 몇 가지 메서드를 사용하였다. String 객체에 관련된 메서드 이다. juminNo.length() 메서드는 문자열의 길이를 반환한다. 예를 들어 juminNo에 “abc” 문자열이 들어 있다면 juminNo.length() 메서드를 호출하면 3이란 숫자를 반환한다. 예제 6.18에서  Member 클래스의 인스턴스 메서드 validJuminNo 내부에 JuminNo라는 로컬 클래스를 생성하였다. 로컬 클래스 JuminNo의 생성자에서는 다음의 코드처럼 String 객체의 length() 메서드를 이용하여 길이가 13자리가 되지 않으면 13자리로 맞추어 반환하였다. juminLength 변수는 13이란 값이 할당이 되어있다.

if(juminNo.length() == juminLength)
this.juminNo = juminNo;
else
this.juminNo = juminNo.replaceAll("-", "");


else 구문 밑의 juminNo.replaceAll() 메서드는 juminNo의 모든 문자 중에서 앞 인자의 문자를 찾아서 뒷 인자로 바꾸어 주는 역할을 한다. 즉 751111-1234567 이런 14자리 문자가 입력되면 7511111234567 이렇게 13자리로 바꾸어 주는 역할을 한다.
로컬 클래스를 만들고 난 뒤에 다음의 코드와 같이 로컬 메스드의 생성자를 이용하여 유효한 주민번호 13자리를 반환한다.

JuminNo jumin = new JuminNo(juminNo);
return jumin.getJuminNo();

validJuminNo 클래스에서 JuminNo 객체를 생성하고 이 객체의  생성자에 인자로 받은 juminNo String 객체를 그대로 전달하였다. JuminNo 클래스 생성자에서 juminNo 문자열의 길이가 13자리인지 체크하여 13자리이면 JuminNo 클래스의 인스턴스 변수에 할당하고 아니면 String 객체의 replaceAll 메서드를 이용해서 “-” 문자를 제거한 뒤에 JuminNo클래스의 인스턴스 변수 juminNo에 할당 하였다. 그리고 validJuminNo 메서드의 마지막에 생성된 JuminNo 객체의 getJuminNo() 메서드를 이용하여 값을 반환 하였다.

final int juminLength=13;

여기서 final 키워드가 선언이 되어있다.  final 키워드를 빼고 위의 소스를 컴파일 하려 하면 컴파일 에러가 발생한다. final 키워드는 상수를 의미한다. 로컬 클래스를 생성하기 전에 로컬 클래스에서 사용해야 하는 값이 고정이 되어 있어야 하기 때문에 컴파일러는 final 키워드가 없으면 에러를 발생 시킨다.
로컬 클래스는 이너클래스와 마찬가지로 static으로 선언된 클래스 변수나 메서드를 가질 수 없다.  하지만 예외적으로 static으로 선언되고 final 키워드를 같이  선언한 상수는 가질 수 있다.  예제 6.18을 조금 변형한 예제 6.19를 살펴보자.

[예제 6.19]
public class Member {

6.4.3	익명 클래스
익명 클래스는 코드를 좀 더 간결하게 만들어 준다. 익명 클래스를 이용하면 클래스 선언과 초기화를 한번에 할 수 있다.  익명 클래스는 이름을 사용하지 않는 점을 제외하고는 로컬 클래스와 동일하다.  예제 6.20은 대표적인 익명 클래스의 사용 예제이다.

[예제 6.20] 익명 클래스
public class Hanger {
public interface Clothes{
public void printColor();
public void printSize();
}
int hangerCount = 1;

public void printClothes(){
final boolean isPrint = false;
// boolean isPrint = false; // 이렇게 하면  컴파일 에러가 발생한다.
Clothes pants = new Clothes(){
public void printColor() {
System.out.println("red pants");
System.out.println(hangerCount);
}
public void printSize() {
System.out.println("32 size pants");
System.out.println(isPrint);
}
};
pants.printColor();  pants.printSize();
Clothes shirts = new Clothes(){
public void printColor() {
System.out.println("red shirts");
}
public void printSize() {
System.out.println("30 size shirts");
}
};
shirts.printColor();  shirts.printSize();
}

public static void main(String[] args) {
new Hanger().printClothes();
}

}


예제 6.20은 대표적인 익명 클래스의 사용 예제이다. Hanger  클래스 내부에 Clothes 인터페이스를 선언하였다. Clothes 인터페이스를 익명 클래스로 구현하여  shirts 와 pants 변수에 할당하였다. 다음의 코드는 우리가 예제 6.20에서 선언하였던 익명 클래스이다.


Clothes shirts = new Clothes(){
public void printColor() {
System.out.println("red shirts");
}
public void printSize() {
System.out.println("30 size shirts");
}
};



익명클래스를  선언하는 방법은  객체를 생성하는 모습과 유사하다. 단지 틀린점은 생성자 뒤에 괄호 (  { } ) 가 있다는 점이다. 괄호 내에 실제 클래스의 구현 내용을 기재하고 있다.
익명 클래스를 구현하기 위해서는 new 연산자가 있어야 하고 인터페이스명과 상속받을 클래스명이 필요하다. 여기서는 Clothes 인터페이스를 만들고 이를 구현하였다. 그리고 생성자와 같이 괄호 안에 매개변수가 들어갈 수 있다. 매개변수가 없다면 빈 괄호( “()”)를 사용해야 한다. 마지막으로 몸체가 필요하다. 익명 클래스의 몸체는 대괄호( { } )로 지정한다.

예제 6.20에서 pants 익명 클래스를 살펴보자. printColor() 메서드에서 System.out.println(hangerCount); 구믄으로 클래스 멤버 변수인 hangerCount 를 출력하였다. 익명 클래스에서 클래스 멤버변수(클래스 인스턴스 변수)에 접근해도 컴파일 에러를 발생시키지 않는다. 두 번째로 printSize() 메서드를 살펴보자 isPrint 변수를 System.out.println(isPrint); 구믄을 이용해서 출력하였다. isPrint 메서드의 로컬 변수이다. 로컬 변수는 익명 클래스 사용하기 전에 final 키워드를 이용하여 상수화 시켜줘야 한다. 예제 6.20에서 final 키워드를 빼면 컴파일 에러가 발생한다.
익명클래스에도 로컬클래스와 마찬가지로 클래스 내부에 static 키워드를 가진 메서드를 선언할 수 없지만 static 키워드를 가진 final 상수는 선언할 수 있다.

익명 클래스는 주로 GUI에서 이벤트 처리시에 많이 사용된다. 특히 안드로이드 프로그램을 하다 보면 화면의 터치나 클릭 이벤트, 키보드 입력 이벤트를 받을 때 익명 클래스를 이용해서 처리한다. 여기서 익명 클래스의 기본 개념에 대해 확실히 집고 넘어가자.

6.5	Enum 타입
Enum 타입은 사전에 정의된 상수의 집합과 같다.  Enum을 사용할 때 변수들은 사전에 정의된 값 중에 한 개와 반드시 일치해야 한다. 일반적으로 Enum을 실무에서 사용할 경우는 주, 방위, 색 배열, 상태정보와  같은 곳에 주로 이용한다. 백문이 불여일타! 직접 예제코드를 통해서 Enum의 사용 예를 살펴보자.


[예제 6.21] Enum의 사용 예제

public class Canvas {
public enum Color{
RED, ORANGE, YELLOW, BLUE, PURPLE
}

Color color;
public Canvas(Color color){
this.color = color;
}

public void whatColor(){
switch(this.color){
case RED:
System.out.println("빨간색"); break;
case ORANGE:
System.out.println("주황색"); break;
case YELLOW:
System.out.println("노란색"); break;
case BLUE:
System.out.println("파란색"); break;
case PURPLE:
System.out.println("보라색"); break;
}
}
public static void main(String[] args) {
new Canvas(Color.RED).whatColor();
new Canvas(Color.ORANGE).whatColor();
new Canvas(Color.YELLOW).whatColor();
new Canvas(Color.BLUE).whatColor();
new Canvas(Color.PURPLE).whatColor();
}

}


예제 6.21의 예제에서 우리는 지금까지 보지 못했던 형태의 키워드를 보았다. 바로 enum 키워드이다. 자바에서 enum 키워드는 Enum을 선언하기 위한 키워드이다. 클래스를 선언하는 모습과 유사한 형태로 선언한다. 다음은 enum을 선언하는 키워드이다.

public enum Color{
// 구현 내용
}

구현 내용 부분에 필드를 정의할 수 있다. 필드는 대문자로 정의해야한다. 두 개 이상의 문자가 올 경우에는 언더버(_)로 연결하여 사용한다. 자바에서는 enum 타입이라고 하고 enum 의 몸체에는 메서드와 필드를 가질 수 있다.  컴파일러는 enum을 컴파일 할 때 자동으로 몇 가지 유용한 메서드를 같이 포함시켜 만든다.  enum 메서드 중에 클래스 메서드인 values() 메서드가 있는데 이는 enum에 포함된 변수들을 배열 형태로 반환한다.  이를 향상된 for 문에서 사용하면 간결하게 코드를 만들 수 있다.

enum은 생성자와 메서드를 모두 가질 수 있다.  eunm의 생성자는 패키지 내부에서만 접근할 수 있다. 하지만 생성자를 우리가 직접 호출 할 수는 없다. 다음을 예제를 통해서 enum 생성자를 만드는 방법을 배워보자.

[예제 6.22] 생성자를 가진 enum
public enum MemberStatus {
APPROVAL_READY("R"), APPROVAL("A"), WITHDRAW("W");

private String statusCode;

private MemberStatus(String code){
statusCode = code;
}

public String getStatus(){
return statusCode;
}

public static void main(String[] args){
for(MemberStatus s : MemberStatus.values()){
System.out.println(s.getStatus());
}
}
}


예제 6.22에서는 생성자를 가진 enum 클래스를 만들었다. APPROVAL_READY, APPROVAL, WITHDRAW 필드를 가지고 있고, 생성자를 가지고 있다. 생성자인 private MemberStatus(String code)는 상태코드를 인자로 가지고 있고 상태코드를 인스턴스 변수인 statusCode에 할당한다.

APPROVAL_READY("R"), APPROVAL("A"), WITHDRAW("W");

위의 구문에서 컴파일러는 자동으로 생성자를 호출하여 “R” 값을 APPROVAL_READY의 statusCode에 할당하고 “A”를 APPROVAL의  statusCode에 할당한다.

마지막으로 main 메서드에서 보면 우리는 MemberStatus enum 클래스에 values() 메서드를 만들지도 않았는데 사용할 수 있다. 컴파일러는 클래스 메서드로 values()를 자동으로 만들어 주었다.  MmebrStatus enum 클래스에 들어가 있는 값을 System.out.println으로 출력해보면 다음과 같은 출력 결과를 얻을 수 있다.

-- 출력 결과 --
R
A
W



6.6	요약

지금까지 6장에서 많은 내용을 공부하였다.  클래스의 기본 개념과 객체를 생성하는 방법을 배웠고 클래스의 선언하는 방법, 객체를 생성하고 사용하는 방법을 배웠다. 클래스는 일종의 설계도이고 객체는 설계도를 바탕으로 만들어진 인스턴스이다. 클래스에는 필드와 메서드를 가질 수 있었다. 필드는 상태정보 메서드는 행위에 대한 정보를 담고 있다. 또한 생성자를 가지고 있어 객체를 생성할 때 초기화할 수 있는 기능도 제공하였다. 필드나 메서드는 접근제어자를 가지고 있어서 외부나 내부에서 접근할 수 있는 범위도  설정할 수 있었다. 현재의 객체를 참조하기 위해서는 this 키워드를 사용하였고, static 키워드를 사용해서 클래스 메서드나 변수를 선언할 수 있었다.
클래스 내부에 클래스를 생성하는 방법도 공부하였다. 중첩 클래스라고도 부르는데 중첩 클래스에는 이너 클래스, 로컬 클래스, 익명 클래스 등이 있었고 중첩 클래스의 내부에서는 메서도에  static 키워드를 사용할 수 없었고 로컬 클래스나 익명 클래스에서 로컬 변수를 참조하기 위해서는 로컬 변수가 반드시 final 로 지정되어 있어야 했다.
마지먹으로 특별한 객체 타입인 enum 타입도 공부하였다. enum  타입은 상태 정보나 색, 요일과 같은 일정한 상수형태의 배열을 제공하고 있고 생성자와 메서드도 가질 수 있었다.

지금까지 객체 지향언어에서 가장 중요하고도 어려운 관문 하나를 넘었다. 조금 내용이 어려울 수 있는 부분이 있지만 모두 완벽히 이해하지 못했다고 해서 걱정할 필요는 없다. 필자는 프로그래밍 언어를 공부할 때는 왕도가 없다고 생각한다. 필자가 강조한 대로 백문이 불여일타! 직접 소스코드를 만들어보고, 실행해보고, 바꾸어보고, 테스트해보고, 디버깅 하면서 실력이 는다고 생각한다. 현장의 많은 자바 개발자들도 마찬가지의 길을 겪었다. 이해가 안되는 부분이 있다면 이클립스나 텍스트 편집기, 도스창을 띄우고 소스코드를 만들고 컴파일 해보자.
