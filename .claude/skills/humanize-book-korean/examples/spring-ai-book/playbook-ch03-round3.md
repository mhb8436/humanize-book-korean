ch03을 사용자 검토하면서 v0.10 humanize 후에도 잔류한 영어 번역체 3종을 신규 등록한 라운드입니다. 본 라운드의 본질은 "Claude가 ch03 800줄을 직접 한 문장씩 검토했음에도 자기 평가·명사화·'묶이는' 영어 직역을 통과시킨 사례". 자기 검토의 구조적 한계를 보여주는 케이스.

## 사용자 검토 사례

원문 (v0.10 humanize + 60건 정리 후):

> 당장은 두 빈의 빌드 결과가 거의 같습니다. 두 빈으로 분리하는 이유는 후속 챕터를 위한 준비입니다. 5장에서 민원인용 ChatClient에는 "안전성 우선 Advisor"가 추가되고, 민원 공무원용 ChatClient에는 "감사 로그 Advisor + 공문 형식의 Advisor"가 추가됩니다. Advisor 체인은 ChatClient 빌더에 묶이는 immutable 구조라서, 사용자 유형마다 다른 빈으로 분리하는 방식이 가장 정리된 형태입니다.

사용자: "이 문장도 니가 직접 검토해줘"

Claude 검토 결과 (자기 비평):
- "당장은" — 일상체 어휘. 사용자 자바책 reference에 0건. **수정**: "이 장에서는"
- "후속 챕터를 위한 준비입니다" — 영어 "preparation for future chapters" 명사화 직역. **수정**: "5장에 있습니다"
- "묶이는 immutable 구조라서" — 영어 "bound to" 직역. **수정**: "한 번 등록되면 바꿀 수 없는 immutable 구조"
- "가장 정리된 형태입니다" — 영어 "the cleanest form" 자기 평가. 한국·일본 IT 책 안 씀. **수정**: 자기 평가 제거, 인과로 환원

수정 후:

> 이 장에서는 두 빈의 빌드 결과가 거의 같습니다. 두 빈으로 나누는 이유는 5장에 있습니다. 5장에서 민원인용 ChatClient에 "안전성 우선 Advisor"가 추가되고, 민원 공무원용 ChatClient에 "감사 로그 Advisor + 공문 형식의 Advisor"가 추가됩니다. ChatClient 빌더의 Advisor 체인은 한 번 등록되면 바꿀 수 없는 immutable 구조입니다. 사용자 유형별로 부착할 Advisor가 다르므로 빈도 사용자 유형별로 분리합니다.

## 메타 질문 — Claude의 자기 검토 한계

사용자: "분명 ch03 모든 문장을 네가 직접 모두 검토해서 점검했는데 여전히 ch03 에서 니가 보기에도 이상한 문장이 계속 나오는 이유는?"

자기 분석:

1. **카테고리 기반 검토의 한계**
   - 미리 정해진 7개 카테고리(한 줄 강조·자문자답·메타 요약·사물 의인화·학습 약속·도식 메타·~다체)로만 매칭
   - 7개에 안 들어가는 패턴은 "OK"로 통과
   - "가장 정리된 형태"는 7개 어디에도 없어서 놓침

2. **"있는 것"보다 "없어야 할 것" 찾기가 어려움**
   - "두 빈으로 분리하는 이유는 후속 챕터를 위한 준비입니다"의 문제는 영어 단어가 아니라 한국어 명사화("준비")
   - 한국어 단어 옷을 입은 영어 명사화 구조는 검출이 어려움

3. **AI가 만든 문장은 AI 자신의 톤 기준에 가까움**
   - LLM 학습 데이터는 영어 IT 책 비중이 압도적
   - 영어식 한국어가 자기에게 "자연스러운 IT 책 톤"으로 잘못 인지됨
   - 자기가 만든 톤을 자기가 검토 → 자동 통과

4. **카테고리 정의 자체가 미완**
   - book-extra-rules.md의 X-1~X-28은 사용자가 짚어준 사례 기반
   - 짚지 않은 패턴은 카테고리에 없음 → 머리의 체크리스트에도 없음

5. **사용자 자바책 reference와 매번 비교 안 함**
   - CLAUDE.md 규정이지만 60건 일괄 정리 중 시간 절약을 위해 reference 비교 생략
   - 사용자 자바책에는 "가장 정리된" 같은 자기 평가 어휘 0건. 비교했으면 잡혔음

비대칭: 사용자는 한국어 원어민 → "한국 IT 책에서 안 보는 표현"을 즉시 인지. Claude는 두 언어 학습 데이터 → 영어식 한국어가 자연스럽게 느껴짐.

## 신규 카테고리 정의

### X-29 자기 평가 형용사

```
정규식: 가장\s*(정리된|깔끔한|단정한|자연스러운|우아한|이상적인|효율적인|적절한)\s*(형태|방식|구조|패턴|선택)
```

**대표 사례:**
- "가장 정리된 형태입니다" → 자기 평가 제거, 인과로 환원
- "가장 자연스러운 패턴입니다" → 사실 진술
- "가장 깔끔한 방식입니다" → 제거

영어 "the cleanest form / the most elegant approach" 직역. 한국·일본 IT 책에 자기 평가 어휘 0건 (책 표지·서평은 출판사 영역).

### X-30 명사화 영어 직역

```
정규식: (은|는|이|가)\s*[^.\n]*?를?\s*위한\s*(준비|기반|토대|발판|초석)\s*입니다
```

**대표 사례:**
- "X은 후속 챕터를 위한 준비입니다" → "X은 5장에 있습니다 / 5장에서 차이가 보입니다"
- "X은 다음 학습을 위한 기반입니다" → "X은 다음 학습에서 사용합니다"

영어 "X is preparation for Y / X is the foundation for Y" 명사화 직역. 시점·이유를 직접 가리키지 못하고 명사화로 우회. 한국 책은 시점을 직접 가리킴.

### X-31 "X에 묶이는" 영어 직역

```
정규식: (빌더|체인|컨테이너|컨텍스트)에\s*묶이는
```

**대표 사례:**
- "빌더에 묶이는 immutable 구조" → "빌더의 Advisor 체인은 한 번 등록되면 바꿀 수 없는 immutable 구조"
- "체인에 묶이는 객체" → "체인에 한 번 부착된 객체"

영어 "bound to" 직역. "묶이다" 자체가 영어 bind 직역 톤이라 평이한 동사로 교체.
