# 공공 SI·eGovFrame 도메인 용어 (Spring AI 책 사례)

본 파일은 공공 SI / 행정 자동화 / eGovFrame 도메인의 한국 표준 용어와 흔히 잘못 쓰이는 표현을 정리합니다. 다른 도메인 책에서는 `examples/{도메인}/glossary.md`를 만들어 자기 도메인 용어를 정리하세요.

## 1. 사업·계약 용어

| 잘못 쓴 표현 | 한국 공공 조달 표준 | 의미 |
|------|------|------|
| **사업소** | **사업자** ✓ | 공공 SI 사업을 수주한 기업·팀 |
| | 수주사 / 수행사 | 같은 의미, 대체 표현 |
| **사업주** | 사업자 / 발주처 | 문맥에 따라 선택 |
| 거래처 | **발주처** ✓ | 공공기관 (시청·구청·중앙부처) |
| 클라이언트 | 발주처 / 사용자 | 의미에 따라 |

**예외 (사람·부서 책임)**: "발주처가 책임을 진다", "운영팀이 장애 대응 책임을 진다" 같은 표현은 자연스러우므로 보존.

## 4. 망분리 환경 용어

| 표현 | 의미 | 사용 위치 |
|------|------|------|
| **망분리** ✓ | 외부 인터넷과 분리된 폐쇄 네트워크 (한국 표준) | 본문 자유 사용 |

## 8. 자주 만나는 잘못된 표현 (한국 IT 책 일반)

본 카테고리는 도메인 무관. 모든 한국 IT 책에 공통.

| 잘못 쓴 표현 | 한국 표준 |
|------|------|
| 끌려 들어와 (is pulled in) | 의존성에 자동으로 추가됩니다 |
| 갈아끼우다 (swap out) | 바꿉니다 / 교체합니다 |
| 한 번에 정렬된 채로 들어와 있다 | 함께 추가되며 같은 버전을 사용합니다 |
| 손에 잡힌다 | 이해됩니다 / 압니다 |
| 손을 코드 위에 올린다 | 코드를 작성합니다 |
| 머릿속에 잡아 둔다 | 이해해 둡니다 |
| RFP가 떨어진다 | RFP가 들어옵니다 / 도착합니다 |

## 사용자가 자기 책에 맞게 수정하는 방법

1. 본 파일을 `examples/{본인 책 도메인}/glossary.md`로 복사
2. 자기 책 도메인의 표준 용어로 교체
3. 잘못 쓰기 쉬운 표현 → 한국 표준 매핑 추가
4. humanize-book 스킬이 본 파일을 자동 참조

도메인 예시:
- examples/spring-ai-book/ (본 사례, 공공 SI)
- examples/ai-engineering-book/ (다른 책, 일반 AI 엔지니어링)
- examples/devops-book/ (DevOps 도메인)
- examples/data-engineering-book/ (데이터 엔지니어링)

각 도메인 폴더에 자기 책 특화 용어집을 두면 됩니다.
